### 3. Hierarchical hypothesis tree as research methodology

> "build out this hypothesis tree... of theories and models of the brain... use AI to falsify every single part of this that is falsifiable, and see what's left"

This is a new methodological idea — a systematic approach to brain theory via hierarchical hypothesis enumeration and falsification. No existing note covers this.

**CREATE** [[writing inbox/neuro/Hierarchical hypothesis tree as research methodology]]

Draft:
```
---
lastmod: 2026-03-09
tags:
  - neuroscience
  - how-do-we-learn
---
Build out a hierarchical tree of theories/models of the brain (starting with C. elegans). For each node, design a minimal experiment that would falsify it. Use AI to run the data analysis; do the hypothesis generation yourself.

This is a way to systematically map the space of what we think the brain is doing, and prune it. If everything gets pruned, you need new theory.

The approach:
1. Enumerate competing theories/models at the right level of abstraction
2. For each, identify falsifiable predictions
3. Design minimal experiments (ideally using existing data)
4. Use LLMs for data analysis/methodology execution
5. Prune what's falsified, see what survives

This intersects with:
- [[What does it mean to understand the brain?]] — the tree is an operationalization of "understanding"
- [[Four components of a brain theory - initializations, cost function, plasticity, learning rules]] — the tree should be organized around these components
- [[Treat answers as tools]] — each hypothesis is a tool, not a commitment
- [[Develop short feedback loops with reality]] — the LOOPER aliveline showed ~2 weeks can test one branch

Open questions:
- What is the right level of abstraction for the tree?
- Can existing public datasets support most of the falsification experiments?
- Is C. elegans the right starting organism?
```
