Build a hierarchical tree of theories/models of the brain. At each node, design minimal experiments to falsify that branch. Use AI to execute the data analysis. Prune what's false. See what's left.

This is enabled by AI — you couldn't search this space manually because each branch requires significant analytical effort. With LLMs handling methodology/data analysis, a single person can speed-run through many branches.

The LOOPER work showed ~2 weeks to test extending a paper's results. At that rate, you could test dozens of hypotheses in a year.

If everything gets pruned, you need a new theory. That's also a useful outcome.

Starting point: [[Four components of a brain theory - initializations, cost function, plasticity, learning rules]] provides the top-level structure. Each component branches into competing hypotheses. Related: [[What mathematical language is appropriate for describing brain computation?]], [[Locomotion is implemented as low-D loops in C elegans]]

This is essentially [[Treat answers as tools]] applied to research methodology — hypotheses are tools for exploring, not positions to defend. And it's [[Develop short feedback loops with reality]] at scale.
